# Project Context

## Project

`qa-learning-lab` is a personal QA engineering practice repository.

The goal is to build a visible learning lab for manual and automated testing skills while keeping the repository useful as a portfolio artifact.

## Main Goals

- Practice Playwright UI automation.
- Practice API testing with Playwright request context.
- Practice SQL for QA reporting and backend validation.
- Keep reusable QA utilities and helpers.
- Practice CI/CD with GitHub Actions, GitLab CI, Jenkins, Docker, and Kubernetes examples.
- Maintain regular GitHub/GitLab activity with meaningful small commits.

## Tech Stack Focus

- JavaScript
- TypeScript
- Playwright
- SQL
- Docker
- GitHub Actions
- GitLab CI
- Jenkins
- Kubernetes
- YAML

## Current Repository Shape

- `api/` contains API tests, API clients, payloads, schemas, and API helpers.
- `playwright/` contains UI tests, page objects, and Playwright fixtures.
- `sql/` contains practice datasets and reporting queries.
- `utils/` contains reusable JavaScript and TypeScript QA utilities.
- `.github/workflows/` contains GitHub Actions workflows.
- `.gitlab-ci.yml` contains GitLab CI pipeline configuration.
- `k8s/` contains Kubernetes manifests.
- `docker/` contains Docker-related runner examples.
